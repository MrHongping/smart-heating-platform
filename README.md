# 智能温控加热台系统

## 项目概述

本项目基于 ESP32 微控制器，开发一套用于加热板的智能温控系统。系统通过温度传感器实时采集温度数据，采用 PID 算法进行闭环控制，并通过 SSR（固态继电器）控制加热功率。同时提供本地显示（I2C 屏幕）和 Web 页面远程控制能力。

系统具备稳定性高、控制精度高、可扩展性强的特点。

## 系统目标

- 实现稳定温控（误差 ≤ ±1℃）
- 支持目标温度设置与实时调节
- 支持 Web 页面远程控制
- 支持本地显示运行状态
- 支持 PID 参数动态调整
- 具备基础安全保护机制（过温保护）

## 硬件组成

系统包含以下硬件组件：

- **主控**：ESP32
- **加热执行**：SSR（控制加热板）
- **加热源**：加热板（支持 AC 或 DC）
- **温度采集**：
  - 优先支持 PT100 + MAX31865（SPI）
  - 兼容 NTC（ADC）
- **显示模块**：I2C OLED（SSD1306）
- **网络**：WiFi（ESP32内置）

## 系统功能

### 1. 温度采集
- 支持周期性采样（周期 100ms）
- 对采样数据进行滤波（滑动平均）
- 转换为实际温度值（单位：℃）
- 提供当前温度数据给控制系统

### 2. 温控算法
- 实现 PID 控制算法
- 输入：当前温度、目标温度
- 输出：加热功率（0~100%）
- 支持参数 Kp / Ki / Kd 可调
- 积分项需限制（防止积分饱和）
- 输出需限制在 0~100%

### 3. 加热控制（SSR）
- 采用时间比例控制（Time Proportional Control）
- 设定固定时间窗口（1 秒）
- 根据 PID 输出占比控制 SSR 开关时间

### 4. 显示功能（I2C）
- 显示内容：当前温度、目标温度、加热状态、PID 输出百分比
- 刷新频率：500ms

### 5. Web 控制功能
- 提供 HTTP 服务，用于远程控制与监控
- API 接口：
  - GET /api/status - 获取系统状态
  - POST /api/set_temp - 设置目标温度
  - POST /api/set_pid - 设置 PID 参数
- WebSocket 实时推送温度数据（1Hz）

### 6. 系统状态管理
- 维护统一状态结构
- 支持多任务访问（线程安全）
- 使用 FreeRTOS 机制（Mutex）

### 7. 多任务调度（FreeRTOS）
- 温度采集任务（100ms）
- PID 控制任务（100ms）
- SSR控制任务（1s窗口）
- 显示任务（500ms）
- Web服务任务（常驻）
- 安全保护任务（500ms）

### 8. 安全机制
- 过温保护：超过设定最大温度（250℃）立即关闭 SSR
- 传感器异常检测：温度异常（断线、异常值）停止加热
- 看门狗机制：防止系统卡死

## 环境要求

- **ESP-IDF**：v5.0 或更高版本
- **编译器**：xtensa-esp32-elf-gcc
- **开发板**：ESP32 系列开发板

## 安装与编译

1. **克隆项目**
   ```bash
   git clone <项目地址>
   cd Smart-heating-platform
   ```

2. **配置 ESP-IDF 环境**
   ```bash
   # 确保 ESP-IDF 环境已正确配置
   export IDF_PATH=<ESP-IDF安装路径>
   ```

3. **编译项目**
   ```bash
   idf.py build
   ```

4. **烧录到 ESP32**
   ```bash
   idf.py flash
   ```

5. **查看串口输出**
   ```bash
   idf.py monitor
   ```

## 使用方法

### 本地控制
- 系统启动后，OLED 屏幕会显示当前温度、目标温度、加热状态和 PID 输出
- 可通过 Web 页面进行远程控制

### Web 控制
1. 连接到 ESP32 创建的 WiFi 网络（默认 SSID：Smart-Heating）
2. 打开浏览器，访问 `http://192.168.4.1`
3. 在 Web 页面上：
   - 查看实时温度数据
   - 设置目标温度
   - 调整 PID 参数
   - 监控系统状态

## API 接口

### 获取系统状态
```
GET /api/status
```

**返回示例**：
```json
{
  "current_temp": 185.3,
  "target_temp": 200,
  "power": 65,
  "heating": true
}
```

### 设置目标温度
```
POST /api/set_temp
```

**请求示例**：
```json
{
  "target": 200
}
```

### 设置 PID 参数
```
POST /api/set_pid
```

**请求示例**：
```json
{
  "kp": 10,
  "ki": 0.5,
  "kd": 50
}
```

## 安全注意事项

1. **过温保护**：系统会在温度超过 250℃ 时自动关闭加热
2. **传感器异常**：当检测到温度传感器异常时，系统会停止加热
3. **电源安全**：确保 SSR 与加热板的功率匹配
4. **散热**：确保系统有良好的散热条件

## 故障排除

1. **无法编译**：检查 ESP-IDF 环境是否正确配置
2. **温度采集异常**：检查温度传感器连接是否正确
3. **加热不工作**：检查 SSR 连接和控制引脚
4. **Web 页面无法访问**：检查 WiFi 连接和 IP 地址
